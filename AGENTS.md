# Repository Guidelines

## Project Structure & Module Organization
- Solution entry: `APISwitch.sln`.
- App source lives in `src/APISwitch/`:
  - `App.xaml(.cs)`: startup and tray lifecycle.
  - `MainWindow.xaml(.cs)`: main UI, tab switching, provider actions.
  - `Views/ProviderDialog.xaml(.cs)`: add/edit provider dialog.
  - `Services/`: `DatabaseService` (SQLite) and `ConfigWriterService` (Codex/Claude config writes).
  - `Models/Provider.cs`: provider entity.
  - `Assets/`: icons and preview assets.
- `Release/` is packaging output (generated by `repack.bat`), not source.
- `openspec/` stores proposals/spec/tasks; update it when requirements change.

## Build, Test, and Development Commands
- `dotnet restore APISwitch.sln` - restore NuGet packages.
- `dotnet build APISwitch.sln` - build all projects.
- `dotnet run --project src/APISwitch/APISwitch.csproj` - run locally.
- `repack.bat` - production repack: restore, publish (`win-x64`, single-file, non-self-contained), output to `./Release`, then clean `src/APISwitch/bin` and `obj`.

## Coding Style & Naming Conventions
- Language: C# (.NET 8, WPF), 4-space indentation, UTF-8 files.
- Naming: `PascalCase` for types/methods/properties, `_camelCase` for private fields.
- Keep UI changes in XAML when possible; place interaction logic in existing code-behind files.
- Prefer focused, minimal diffs; do not refactor unrelated code in feature PRs.

## Testing Guidelines
- No dedicated test project yet; use build + manual smoke tests.
- Minimum checks before merge:
  - add/edit/delete provider
  - activate provider and verify config writes
  - Codex/Claude tab behavior
  - tray show/hide/exit behavior
  - open-config-directory button behavior
- If adding automated tests, create `tests/APISwitch.Tests` and use descriptive names like `ActivateProvider_ShouldToggleSingleActiveItem`.

## Commit & Pull Request Guidelines
- Current history favors short, action-oriented messages (for example: `fix UI`, `打开配置目录`).
- Recommended format: `<type>(scope): summary` (e.g., `fix(main-window): disable activate button for active provider`).
- PRs should include:
  - what changed and why
  - risk/impact notes (especially config-writing changes)
  - UI screenshots/GIFs for visible changes
  - manual verification steps.

## Security & Configuration Tips
- Never commit API keys, local user config files, or generated `Release` binaries unless explicitly required.
- Runtime paths used by app:
  - `~/.codex/`
  - `~/.claude/`
  - `~/.APISwitch/apiswitch.db`.
