## Context

当前供应商列表已有 `SortOrder` 字段并按升序展示，但缺少用户可见的顺序调整交互。需求是通过上移/下移按钮在当前标签页（ToolType 维度）内快速重排，并持久化。

## Goals / Non-Goals

**Goals:**
- 提供卡片级“上移(↑)/下移(↓)”操作
- 仅在当前 ToolType 列表内重排
- 顺序调整后立即持久化并刷新 UI
- 边界项禁用对应按钮，防止无效点击

**Non-Goals:**
- 不实现拖拽排序
- 不新增数据库字段
- 不改变跨 ToolType 的相对顺序策略

## Decisions

### 1. 重排算法使用“相邻交换”
- 上移：与前一项交换 `SortOrder`
- 下移：与后一项交换 `SortOrder`
- 原因：逻辑简单、事务范围小、易验证
- 备选：全量重排（实现复杂且没必要）

### 2. 数据层新增专用重排方法
- 在 `DatabaseService` 增加 `MoveProviderUp/MoveProviderDown`（或统一 `MoveProvider`）
- 在数据库事务中读取相邻记录并交换 `SortOrder`
- 原因：避免 UI 层直接拼接排序 SQL，职责更清晰

### 3. 边界禁用由 UI 绑定控制
- 首项禁用“上移”，末项禁用“下移”
- 原因：减少无效请求，交互可预期
- 备选：允许点击后提示（体验较差）

### 4. 顺序归一化策略
- 当前阶段不强制每次全量归一化，只做交换
- 若后续发现历史数据异常，再增加“按列表重排归一化”维护操作

## Risks / Trade-offs

- [并发重排导致顺序冲突] → 单用户桌面场景 + 事务交换，风险可接受
- [历史脏数据导致相邻关系异常] → 初版按当前排序工作，必要时补归一化功能
- [按钮增多影响卡片拥挤] → 仅增加两个小按钮，保持同一操作区

## Migration Plan

- 无数据库结构迁移
- 发布后即生效，旧数据直接可重排
